\begin{abstract}
We propose a governance framework and evaluation methodology for LLM-driven
autonomous ML experimentation, providing metrics and tooling for assessing
agent reliability beyond task performance.
Autonomous ML experimentation agents can propose code edits, execute
training runs, and select follow-up trials, but task metrics alone do not
reveal whether those actions were bounded, auditable, failure-aware, or
reproducible.
We argue that governed autonomous experimentation requires a complementary
evaluation layer: a suite of \emph{governance metrics}---acceptance rate,
invalid-action rate, repeated-bad rate, provenance completeness, artifact
capture completeness, and a failure taxonomy---that characterise the
reliability of the experimentation process, not just its output.
We present a governed harness that operationalises this evaluation framework.
In the Agent~=~Model~+~Harness decomposition~\cite{trivedy2026anatomy},
our harness owns trial lifecycle, scope enforcement, and audit; the LLM
manager is a replaceable component.
The framework separates proposal, execution, validation, and decision
authority; enforces editable-scope constraints via a node specification;
records append-only trial ledgers with complete provenance; and exports the
full governance metric suite per campaign.
We validate the framework on the ResNet-trigger ML benchmark with a
five-trial real-worker primary campaign (100\% provenance completeness,
full failure taxonomy, complete artifact capture) and three evaluation
sub-studies: a 10-trial governance diagnostic campaign in which ledger
inspection identified and repaired search-space exhaustion without any
re-run; a fail-safe demonstration showing that \texttt{edit\_failed} trials
are correctly classified as \texttt{failed\_invalid} rather than accepted as
improvements; and a 10-trial stochastic memory ablation showing a
non-monotonic result (summary memory outperforms both no-memory and
rationale-format memory on repeated-bad rate and best AUC).
Task-metric AUC improves modestly (+0.000933) as secondary evidence that
the loop runs correctly; the central result is that lifecycle decisions,
failures, and provenance are jointly auditable for every trial.
\end{abstract}

\section{Conclusion}
\label{sec:conclusion}

We presented a governed control plane for autonomous ML experimentation that
treats trial lifecycle, validity, provenance, and failure handling as
first-class evaluation objects. The central design choice is separation of
authority: managers propose experiments, workers execute edits and capture
artifacts, and the control plane owns scope enforcement, metric parsing,
keep/discard decisions, memory updates, and the append-only ledger. No
component except the control plane can commit trial state or modify the
ledger.

The framework demonstrates four concrete governance properties on the
ResNet-trigger node: (1)~a five-trial real campaign with 100\% provenance
completeness and full failure taxonomy; (2)~fail-safe classification of novel
failure modes, including \texttt{edit\_failed} and
\texttt{proposal\_precondition\_failed}, as first-class auditable records;
(3)~a governance diagnostic cycle in which ledger inspection identified three
root causes of search-space exhaustion without any re-run, followed by a
verified repair; and (4)~a 10-trial stochastic memory ablation showing a
non-monotonic result --- summary-format memory outperforms both no-memory and
rationale-format memory --- with all outcomes fully provenance-tracked.

The central methodological claim is that autonomous experimentation agents
should be evaluated not only by task metrics, but by governance metrics that
expose whether the experimentation process was bounded, reproducible, and
scientifically inspectable. Task metrics alone cannot reveal whether a system
repeatedly attempted invalid edits, concealed failed runs, or accepted an
unedited baseline as an improvement. The harness makes those behaviors visible
by design.
