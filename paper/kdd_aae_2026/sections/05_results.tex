\subsubsection{\texorpdfstring{Arm A: \texttt{prompt\_manager} (deterministic round-robin)}{Arm A: prompt\_manager (deterministic round-robin)}}\label{arm-a-prompt_manager-deterministic-round-robin}

Three arms at equal budgets (5 trials each), differing only in the memory context injected into the proposal selector. Campaign IDs: \passthrough{\lstinline!ablation\_none!}, \passthrough{\lstinline!ablation\_append\_only\_summary!}, \passthrough{\lstinline!ablation\_append\_only\_summary\_with\_rationale!}.

\textbf{Result: flat.} All three arms produced identical proposal sequences, outcomes, and final metrics (2 kept, 0 discarded, 3 failed-invalid, repeated\_bad\_rate = 0.40, best val\_auc = 0.774711). \subsubsection{\texorpdfstring{Arm A3: 20-trial scientific campaign (\texttt{kdd\_resnet\_scientific\_20\_trials})}{Arm A3: 20-trial scientific campaign}}\label{arm-a3-20-trial-scientific}

A 20-trial campaign was run on the ResNet-trigger node using the same structured manager, worker, metric parser, and reset discipline as Arm A2. The longer budget probes search-space behavior beyond the 10-trial horizon.

\textbf{Governance summary:} 4~kept, 5~discarded (\passthrough{\lstinline!degraded\_metric!}), 11~failed-invalid (\passthrough{\lstinline!proposal\_precondition\_failed!}). Best \passthrough{\lstinline!val\_auc!} reached: 0.782733.

\textbf{Key finding: search-space exhaustion at scale.} Eleven trials (55\%) were classified \passthrough{\lstinline!proposal\_precondition\_failed!}~---~all caught before worker invocation, with \passthrough{\lstinline!wall\_clock\_seconds\,=\,0.0} on every rejected trial (zero GPU time wasted). Ledger inspection of the structured \passthrough{\lstinline!extra.manager.skipped\_candidates!} field identified three root causes:

\begin{itemize}
\tightlist
\item \textbf{Static candidate pool exhaustion (approx.\ 7/11).} The proposal selector's candidate list contains one target value per symbol. With \passthrough{\lstinline!RESNET\_TRIGGER\_FAST\_SEARCH=1!} active, only 9 of the 32 global candidates are valid for the ResNet node; after $\sim$10 trials all 9 are marked \passthrough{\lstinline!already\_attempted!} and the selector cannot find a valid remaining edit.
\item \textbf{Environment-override shadowing (approx.\ 3/11).} \passthrough{\lstinline!BATCH\_SIZE!} and \passthrough{\lstinline!EPOCHS!} are pinned by fast-search environment variables regardless of the constant in \passthrough{\lstinline!train.py!}. The effective-config check correctly classifies edits to those symbols as \passthrough{\lstinline!effective\_config\_unchanged!}; the root cause is that the candidate list did not distinguish env-pinned symbols from freely editable ones.
\item \textbf{Stale symbol references (approx.\ 1/11).} Candidates referencing \passthrough{\lstinline!HIDDEN\_DIM!} (a synthetic-node symbol absent from the ResNet \passthrough{\lstinline!train.py!}) are skipped with \passthrough{\lstinline!symbol\_missing!}. The global candidate list was not filtered per node type.
\end{itemize}

These root causes were diagnosed entirely from the append-only ledger without re-running any trial. Two corrective changes were applied: (1) the candidate pool was expanded from 32 to 45 entries by adding multi-magnitude targets per symbol (four \passthrough{\lstinline!LEARNING\_RATE!} values, four \passthrough{\lstinline!WEIGHT\_DECAY!}, four \passthrough{\lstinline!DROPOUT!}, three \passthrough{\lstinline!GRAD\_CLIP\_NORM!}, three \passthrough{\lstinline!KERNEL\_SIZE!}), raising the count of valid ResNet-trigger candidates from 9 to 21; (2) \passthrough{\lstinline!BATCH\_SIZE!} and \passthrough{\lstinline!EPOCHS!} are now filtered upfront with reason \passthrough{\lstinline!env\_overridden!} when \passthrough{\lstinline!RESNET\_TRIGGER\_FAST\_SEARCH=1!} is active.

\textbf{Post-fix follow-up campaign (\passthrough{\lstinline!kdd\_resnet\_scientific\_10!}, $N=10$).} A clean 10-trial campaign was run after the fixes under the same node, manager, and reset discipline. Governance summary: 3~kept, 7~discarded, \textbf{0~failed-invalid}. Acceptance rate: 0.30. Invalid rate: 0.00. Best \passthrough{\lstinline!val\_auc!}: 0.776756. Net gain: +0.002045. \textbf{Non-monotonic memory result.} The pre-stated hypothesis (repeated\_bad\_rate: none $>$ summary $>$ rationale) is not confirmed. The \passthrough{\lstinline!append\_only\_summary!} arm is the clear standout: it achieved the lowest repeated-bad rate (0.50 vs.\ 0.80 for both other arms) and the highest best AUC (0.8212 vs.\ 0.774711). The \passthrough{\lstinline!none!} and \passthrough{\lstinline!append\_only\_summary\_with\_rationale!} arms are tied on both metrics. The non-monotonic ordering --- summary $<$ none = rationale --- suggests that summary-format memory is the most actionable representation for this LLM and task, while the additional verbosity of rationale context does not improve (and may hinder) the LLM's ability to diversify its proposals.

\textbf{Proposal diversity.} The summary arm is also the most structurally diverse. It discovered a new search dimension not attempted by either other arm: \passthrough{\lstinline!TRAIN\_FRACTION!} (T6: 0.7$\to$0.8, T8: repeat keep). The \passthrough{\lstinline!none!} and \passthrough{\lstinline!rationale!} arms both converged on batch-size variants (BATCH\_SIZE 64, 128, or baseline 32) across the majority of trials. This pattern is consistent with the summary context providing compact signal about which proposal classes have already failed, nudging the LLM toward unexplored dimensions.

\textbf{\passthrough{\lstinline!edit\_failed!} governance finding (historical).} The original five-trial LangGraph runs (\passthrough{\lstinline!lg\_ablation\_*!}, prior to the deterministic-patch bridge) exposed a fail-safe property: when the \passthrough{\lstinline!claw\_style\_worker!} could not apply free-form LLM proposals without a live AI coding agent, training executed against the unmodified \passthrough{\lstinline!train.py!}. The control plane correctly classified all 15 of those trials as \passthrough{\lstinline!failed\_invalid / edit\_failed!}~--- it did not accept the baseline metric (val\_auc = 0.774711) as a keep. This demonstrates that the governance layer enforces edit-provenance correctness even when a downstream component is absent. The v2 campaigns (\passthrough{\lstinline!lg\_ablation2\_*!}) eliminate this dependency via the deterministic-patch bridge, producing zero \passthrough{\lstinline!edit\_failed!} outcomes across 30 trials.

\subsection{Task Metric Trajectory}\label{task-metric-trajectory}

This margin is small relative to baseline seed variation: five clean baseline runs over seeds 123-127 have mean \passthrough{\lstinline!val\_auc=0.784755!}, population standard deviation 0.013837, and bootstrap 95\% CI for the mean {[}0.774809, 0.798329{]}.
